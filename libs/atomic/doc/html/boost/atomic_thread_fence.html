Function atomic_thread_fence

boost::atomic_thread_fence — Insert explicit fence.

Synopsis

// In header: <boost/libs/atomic/doc/atomic.hpp>


void atomic_thread_fence(memory_order order);

Description

Inserts an explicit fence. The exact semantic depends on the type of fence inserted:

  • memory_order_relaxed: No operation

  • memory_order_release: Performs a "release" operation

  • memory_order_acquire or memory_order_consume: Performs an "acquire" operation

  • memory_order_acq_rel: Performs both an "acquire" and a "release" operation

  • memory_order_seq_cst: Performs both an "acquire" and a "release" operation and in addition there exists a global total order of all memory_order_seq_cst operations

Parameters:

order

Memory ordering constraint
